# SEO editorial workflow

Search optimization begins after the page solves the reader's problem. It does not compensate for thin summaries.

## Page-level workflow

1. Name one natural search intent. Book pages target the title plus “summary”; answer pages use the reader's full question.
2. Write a concise, descriptive title that leads with that intent and avoids repeated keywords.
3. Write a unique description that states the page's specific value in 80–200 characters.
4. Answer the intent in the visible opening, then support it with complete, original reasoning.
5. Link source books and adjacent pages with descriptive anchor text inside a meaningful context.
6. Show who edited the page and link to the editorial method.
7. Emit canonical, social, Article, Book where applicable, breadcrumb, and collection structured data.
8. Confirm the page is linked from an index or another relevant page, present in the sitemap, and crawlable without client-side interaction.
9. Run the full content and SEO audits against the built HTML.

## Catalog completion gate

Every book and answer must have:

- a passing strict content score;
- one unique title and description aligned with its intent;
- exactly one H1;
- a visible editor byline;
- valid canonical and social metadata;
- Article structured data plus its source Book entities;
- at least one crawlable incoming catalog link;
- descriptive outgoing source links;
- no invented examples, preview copy, or generic template endings.

The implementation follows Google Search Central's people-first content, title link, snippet, and crawlable-link guidance. The editorial rubric is intentionally stricter than metadata validity alone.
