# Answer editorial workflow and rubric

## Purpose

An answer is a self-contained response to one recurring problem. It uses books as source material, but it is not a collection of book takeaways, a preview of a personalized agent response, or a compressed book digest. The reader should understand what is happening, why it happens, and what to do differently without opening another page.

## Repeatable workflow

### 1. Define the problem precisely

Write the recurring situation in one sentence. Identify the decision the reader cannot currently make and the observable symptom that brought them here. Do not begin drafting until the problem is narrower than a topic.

### 2. Build the source brief

Read every linked book digest completely. Extract only the claims and mechanisms that materially change this problem. Record which book supports each idea. Examples must come from the book or another named source; never invent a fictional worked example.

### 3. State the answer before outlining

Write one or two sentences that answer the question directly. The answer should be non-obvious enough to change the reader's model and specific enough to exclude plausible alternatives. If the thesis is merely “follow these tips,” the source work is not finished.

### 4. Build an argument, not a template

Use four to six idea-led sections. A useful sequence is problem, mechanism, distinction or diagnosis, decision consequence, and next move, but headings should follow the argument rather than repeat those labels. Each section must add a claim, mechanism, source-grounded example, qualification, or implication.

Use paragraphs by default. Lists are reserved for genuinely sequential procedures, compact diagnostic checks, or exact question sets. Add a simple line illustration only when it makes a causal chain, comparison, or decision path easier to grasp.

### 5. Make it usable without making it generic

Give the reader a decision rule or next move that follows from the reasoning. Keep important qualifications beside the claims they modify. Do not append mandatory “boundary,” “related books,” or motivational-summary sections.

### 6. Edit and verify

Cut preview language, repeated claims, ornamental advice, and anything that requires the reader to open the source book to understand the answer. Verify every source link, example, quotation, and illustration. Run the content audit, build, and rendered UI regression test.

## Scored rubric

Score each answer out of 100. Publish at 85 or above, with no hard failure.

| Dimension | Points | Passing evidence |
| --- | ---: | --- |
| Self-contained insight | 20 | The opening answers the question directly and the page makes sense without another source. |
| Problem diagnosis | 15 | The answer distinguishes the underlying failure from its visible symptom. |
| Mechanism and reasoning | 20 | Claims are explained through cause and effect rather than asserted as tips. |
| Source fidelity | 15 | Book ideas are linked, accurately represented, and clearly separated from editorial inference. |
| Decision usefulness | 15 | The reasoning produces an observable decision rule, next move, or update condition. |
| Prose and structure | 10 | Paragraphs substantially outnumber list items; headings carry the argument; length is normally 900–1,400 words. |
| Visual restraint | 5 | Any illustration clarifies a real relationship and remains accessible; no visual is added merely for decoration. |

## Hard failures

Do not publish when any of these are true:

- the page previews an answer instead of delivering it;
- a major claim has no source or reasoning;
- an example is invented or presented without attribution;
- the page is primarily bullet points;
- no linked catalog book materially supports the argument;
- the next move does not follow from the diagnosis;
- generic template sections replace an actual ending.
