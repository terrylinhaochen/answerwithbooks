# Answer cover workflow

Answer covers are generated by `scripts/generate-answer-cover-assets.mjs`. The script owns the palette rotation, answer-specific metaphor, prompt record, dimensions, and output path.

## Generate one cover

Use Gemini/Nano Banana when a valid credential is available:

```bash
npm run answer-covers:gemini -- --ids=answer-slug
```

Use the deterministic vector-to-JPEG renderer when model generation is unavailable:

```bash
npm run answer-covers:local -- --ids=answer-slug
```

Partial runs merge into `public/answer-covers/prompts.json`; they must not erase prompts for other covers.

## Required QA

```bash
npm run answer-covers:check -- --ids=answer-slug
npm run dev
```

Inspect the cover on `/answers/` at its real card size. Accept it only when:

- the metaphor reads in one glance at thumbnail size;
- there is one dominant emblem with no collisions or near-tangencies;
- the left 45% stays calm for the website title overlay;
- the asset contains no generated text, fake glyphs, logos, or UI;
- it looks like the neighboring covers, not merely like the written prompt.

If any check fails, change the metaphor or regenerate. Do not rationalize a malformed result because the prompt was technically followed.
